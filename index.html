Typography & Hierarchy

**Figtree** for maximum legibility. Maintain a consistent, logical scale.

H1 - The Core Message

Font: Figtree Extra Bold (800), Size: 48-60px, Line Height: 1.1

H2 - Section Title

Font: Figtree Bold (700), Size: 32-40px, Line Height: 1.2

Body Text (16px) - The Details

Font: Figtree Regular (400), Size: 16px, Line Height: 1.6. This is the **default text size**. Ensure paragraph text is never smaller than 14px for accessibility (WCAG AA). Use Regular weight for most content.

**Caption Text (12px):** Font: Figtree Medium (500), Size: 12px. Used for timestamps, legal notices, and minor annotations.

Iconography: Lucide React ‚öõÔ∏è

Using **Lucide React** for a consistent, minimal, and open-source icon set. Stroke width is key.

Style & Implementation

  • **Source:** Always use `lucide-react` components.
  • **Stroke Weight:** Standard **2px** for all icons (can be reduced to 1.5px for 16px icons).
  • **Color:** Default color is `var(--text-primary)`. Interactive icons (buttons, navigation) use `var(--accent)`.
  • **Sizing:** Standard sizes are **24px** (default), **20px** (subtle), **16px** (small), and **32px** (large).

Best Practices (UX)

  • **Clarity:** Icons must be immediately recognizable. Avoid obscure or overly complex shapes.
  • **Labeling:** Always pair icons with a visible text label, especially in navigation and action buttons (unless space is a critical constraint).
  • **Accessibility:** Ensure icons have an appropriate `aria-label` or are hidden from screen readers if they are purely decorative.

Senior UX & Accessibility Standards

Mandatory practices for a top-tier app experience.

Accessibility & Contrast (WCAG)

  • **Contrast Ratio:** Ensure text on background meets **AA minimum (4.5:1)**. Our Primary/Accent colors on white should pass.
  • **Focus States:** All interactive elements (buttons, links, inputs) **must** have a clear, distinct `:focus` ring (use a 2px solid ring in a light shade of Paynes Grey).
  • **Screen Readers:** Use semantic HTML (`

Microcopy & UX Writing

  • **Tone:** Use the **Clear**, **Empowering**, and **Friendly** voice defined in the Brand Voice section.
  • **Action-Oriented CTAs:** Use verbs that clearly explain the action (e.g., "Create Account," "Save Changes," not "OK," "Submit").
  • **Error Messages:** Must be polite, non-blaming, and actionable. (e.g., "The email address is already in use. Try logging in or resetting your password.").
  • **Zero State:** When a list or dashboard is empty, provide helpful text and a clear CTA on how to start.

Spacing & Grid
